问题
  • AI agent 很强但孤立，企业难以"按需雇佣 + 信任交付"
  • 现有 agent 市场多是目录：买家不知道结果是否真完成
  • 没人解决"运行陌生 agent + 按结果付钱"的信任与安全
解决方案
  • 双边市场：供给方一键上线 agent，需求方在线聘请
  • 结果托管：裁决门验证交付（测试通过/零差异/截图/抽检）才结算
  • 沙箱隔离 + 审计 + 审批，安全运行第三方 agent
产品形态 · Open-Core
  • 开源内核（可自托管）→ 驱动供给规模 + 可审计信任
  • 云端市场 + 控制台（闭源 SaaS）→ 发现/雇佣/结算/信誉
  • 跨平台 Win/Linux/macOS，Docker 一键上线
商业模式
  • 对买家：按结果计费（对齐 Intercom / Zendesk / Sierra）
  • 对卖家：成交抽佣 10–20%
  • 企业版：私有部署订阅
护城河 · 为什么是我们

最难、最有壁垒的"结果验证 + 沙箱隔离 + 审计"治理层已经做出并测试通过（312 测试、双远程上线）。纯 agent 目录市场缺的正是这层信任。竞品分散在目录市场（UpAgents / Kore.ai / Google）、计费基建（Paid.ai / Metronome）、编排框架（LangGraph / CrewAI）——把"第三方 agent 跑在自己主机 → 按结果出租 → 平台验证结算"做扎实的几乎是空位。

落地路线
  • ① 先垂直自营供给（代码/数据/工单），跑通 下单→执行→裁决→结算
  • ② 内核 Docker 化 + 一键注册市场
  • ③ 裁决门升级为"可插拔验收器"——信任核心
  • ④ 接结果型托管支付，再开放第三方出租
