# 最终方案与下一阶段开发目标（Phase 2）

## 战略定调（一句话）

**先不做开放市场，先用现成内核把"单一垂类的按结果自营交付"跑成正毛利的小生意；用最小成本验证两个生死指标，再决定是否加码到平台与第三方出租。**

- 主路径：**自营按结果交付**（最现实，现在就能赚钱）
- 并行变现：**内核私有部署**（最确定的现金流，竞争最少）
- 暂缓：开放市场 / 第三方出租（等单位经济与信任验证通过后再做）

## 两个生死指标（Phase 2 唯一目标）

整个下一阶段只为回答两个问题。达标→加码；不达标→收手或改模式。

1. **单任务毛利为正**：`结果价 − (模型 token 成本 + 主机成本) > 0`，且批量平均为正。
2. **结果验证准确率 > 90%**：裁决门判"completed"的任务里，人工抽检真正合格的比例 > 90%（防止假完成导致赔付/退款）。

## 必须先堵的风险：按结果付费的成本方差

核心机制改造——**给每个任务设成本闸**，把 token 方差关进笼子：

- 单任务 **token / 时长 / 重试上限**：超限即停，转"受阻→报价"而非继续烧钱。
- **按场景定价**：不同任务类型不同结果价（不要一口价通吃）。
- 难任务走**报价制**（先估算成本再承接），不进固定价池。
- 供给方**自带模型/主机**时，成本由其承担，平台只抽佣——天然对冲方差。

## Phase 2 开发目标（按优先级）

### P0 — 让"按结果"在经济上成立（核心）
1. **成本计量闭环**：在 adapter 执行里记录每任务真实 token / 时长 / 重试次数 → 落 `budget` 表（团队已有 budget record 雏形，需补全 token 计量）。
2. **成本闸 / 上限**：任务级 `max_tokens / max_runtime / max_retries`，超限自动 block 并标记"需报价"。
3. **单位经济报表**：`companyctl economics` —— 按任务类型输出 结果价 / 成本 / 毛利 / 毛利率，控制台出一页看板。

### P0 — 让"结果"可信（护城河）
4. **可插拔验收器（Verifier）**：把裁决门从"读 STATUS 文本"升级为可插拔验收：
   - `test`：跑指定测试/命令，退出码为准
   - `numeric`：关键数字与期望零差异
   - `artifact`：产物文件存在 + 校验
   - `screenshot`：截图比对（人工或视觉模型）
   - `human`：人工抽检队列
   每个任务卡声明用哪种验收 + 通过阈值；裁决以验收器结果为准，不再只信 agent 自述。
5. **抽检与准确率统计**：完成任务进抽检队列，记录"裁决判定 vs 人工复核"，自动算验证准确率。

### P1 — 自营交付闭环 MVP（验证载体）
6. 选 **1 个垂类**（建议：代码任务/测试修复，或数据抓取→报表，结果可机器验证）。
7. 跑通端到端：**下单（含验收标准+预算上限）→ 执行 → 验收器裁决 → 结算记账**，控制台可视化。
8. 真实跑 **20–50 个任务**，采集两个生死指标的数据。

### P1 — 内核私有部署产品化（并行现金流）
9. **Docker 化**：`docker run company-kernel` 一键起内核+网关+控制台。
10. 许可与计量：私有部署版打包 + 简单 license 校验，作为可售 SKU。

### P2 — 暂缓（达标后再做）
- 开放市场前端（发现/雇佣/信誉/仲裁）
- 第三方供给方出租 + 押金/分成结算
- 结果型托管支付（Stripe / Paid.ai 集成）

## 30 天验证计划

| 周 | 目标 | 产出 |
|---|---|---|
| 第 1 周 | 成本计量闭环 + 成本闸 | 每任务能算出真实成本，超限会停 |
| 第 2 周 | 可插拔验收器（先做 test + numeric + human 三种） | 裁决以验收器为准，抽检队列上线 |
| 第 3 周 | 选定垂类 + 自营闭环 MVP + 单位经济报表 | 能下单→执行→验收→看到毛利 |
| 第 4 周 | 跑 20–50 真实任务 + 采集两指标 | **单任务毛利 / 验证准确率 数据出炉** |

## 决策门（第 30 天）

- 毛利为正 **且** 验证准确率 >90% → **加码**：扩垂类、做开放市场、考虑融资。
- 只满足一个 → **改模式**：调价格/换垂类/收紧成本闸，再跑一轮。
- 都不满足 → **收手**：说明"按结果付费"在当前成本结构下不成立，及时止损。

## 为什么现在做成本低、风险小
内核（任务/裁决/沙箱/审计/审批/网关/备份）已完成且 312 测试通过。Phase 2 主要是"加计量 + 加验收器 + 选垂类跑数据"，**不是从零造系统**，所以用很小代价就能证伪或证实这门生意。
