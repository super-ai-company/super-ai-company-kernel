## 这些功能到底怎么用（产品视角）

- **任务看板**：派任务给员工 → daemon 自动喂给 codex 执行 → 裁决门验收。已可用。
- **消息/直连执行**：`仅记录`写收件箱；`直连执行`真实唤起对方运行时（codex/hermes）。已可用。
- **对话**：多个 AI 员工多轮讨论。**当前只能回复已有会话，不能在控制台发起新会话**（你截图卡在这里）——本轮修复。
- **审批**：高风险动作（外发/部署/改内核）进审批中心，owner 一键批准。中文 ID bug 已修。
- **活动流**：adapter 执行记录 + 公司事件，零 token 监控。已可用。

## 升级任务清单（按优先级）

### P0 — 上线硬阻断
1. **网关鉴权**：当前任何能访问端口的人都能派任务/批审批/删员工。需加 token 鉴权（`Authorization: Bearer`）+ 写操作校验 `by` 身份。`--host` 默认 127.0.0.1 是唯一现有防线。
2. **审计可信化**：API 的 `by` 字段来自请求体、不可验证。需把鉴权身份与审计 actor 绑定。
3. **company.sqlite 备份/恢复**：唯一数据源无自动备份。需定时快照 + 一键恢复脚本。

### P1 — 可用性关键
4. **控制台发起新对话**：补"新建会话"表单（选参与者+标题+首条消息）。← 本轮先做
5. **控制台新增员工表单**：UI 调用 `company-add-employee`，免命令行。
6. **控制台任务详情增强**：直接显示 evidence 内容预览、裁决 STATUS、重试历史。
7. **证据落盘统一**：codex 证据写在 controller 工作区（内核外），doctor 无法验证。需强制证据回写内核 `employees/<id>/reports/`。
8. **hermes/claude/antigravity 真实执行认证**：当前仅 dry-run，需各跑通一单真实执行（已派给 codex 设计 runbook）。

### P2 — 体验/规模化
9. **裸机一键引导脚本**：clone→env→launchd→verify 合一。
10. **Cowork 对话式 onboarding skill**：封装 add-employee，自然语言加员工。
11. **移动端响应式**：控制台仅 1 个 media query，窄屏体验差。
12. **实时推送**：当前 15s 轮询，可升级 SSE/WebSocket。
13. **健康噪音治理**：闲置 OpenClaw 员工的 stale 心跳应单独归类，不计入"异常"。
14. **多机部署文档**：REST/gRPC 已支持远程员工，缺部署与网络配置指南。
